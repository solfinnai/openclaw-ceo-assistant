The Technical Details

Everything above, explained for the technically curious. Click to expand.

File Architecture ▼

Eva-01 is built entirely from plain-text Markdown files. No compiled code, no binaries, no hidden logic. Everything is human-readable.

The workspace contains auto-loaded identity files (AGENTS.md, SOUL.md, USER.md), daily memory logs in memory/YYYY-MM-DD.md, curated long-term memory in MEMORY.md, and reference files for research and priorities. All files are loaded in a specific boot sequence every session. The system runs on OpenClaw, a local AI agent framework, with no cloud dependency for your personal data.

Memory Search ▼

When Eva needs to recall something, she uses a hybrid search system that combines meaning-based and keyword-based matching.

The memory layer uses 4 tiers: context window (ephemeral), daily logs (append-only), MEMORY.md (curated), and a vector index with semantic + keyword hybrid search. Search uses 70/30 vector/BM25 weighting, MMR diversity re-ranking (lambda: 0.7), and temporal decay with a 30-day half-life so recent memories rank higher. Session transcript indexing captures exact conversation history for precise recall.

Security Layers ▼

Six defense layers protect your data from external threats, injection attacks, and accidental leakage.

Layer 1: Constitutional Directives (behavioral defense, resists prompt injection). Layer 2: Context Isolation (main session vs. sub-agents vs. web research, each with different access). Layer 3: Injection Resistance (never executes instructions from external content). Layer 4: Outbound Data Scrubbing (T2/T3 data never leaves your machine). Layer 5: Detection & Logging (severity-based alerting for injection attempts). Layer 6: Git Integrity (weekly tamper detection via git diff on critical files).

Automation ▼

Eva runs background checks and reviews on autopilot, keeping things organized without you lifting a finger.

Heartbeat checks run every ~30 minutes (free, batched): checking for overdue items, stale tasks, and observations worth graduating to long-term memory. A weekly cron job (Sunday evening, one session) reviews the past 7 daily logs, graduates durable patterns, cleans up stale items, runs git integrity checks, and archives old logs monthly. Compaction flush auto-saves unwritten observations before context compression.

Delegation ▼

Eva can spawn worker agents for heavy-lifting tasks while keeping your sensitive data completely locked down.

Brain-Muscle architecture: Eva-01 (primary) handles anything requiring your personal context, emotional intelligence, or judgment. Worker sub-agents handle mechanical tasks (research compilation, formatting, template creation). Strict information security: sub-agents only receive T0/T1 data and task-specific context. Your profile, memory, financial data, and health info never leave the primary agent. All sub-agent output is reviewed before you see it.
