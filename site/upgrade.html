Upgrade Guide

Upgrade Your Agent

A major upgrade in about 5 minutes. Your agent does all the hard work. You just hand it the files.

Your Data Stays Safe

All your personal stuff (profile, memories, daily logs, priorities) is untouched. The upgrade only changes the operating system, not your data. Your agent's personality, your memories, your daily logs, everything that makes it yours stays exactly where it is.

Step-by-Step

The Upgrade Process

8 simple steps. Copy the prompts, paste them to your agent, let it do the work. That's it.

0

Back Up Your Current Agent First

Before you upgrade anything, make a full backup. That way, no matter what happens, you have a complete snapshot you can restore from. Do this first.

Before we do anything, I need a complete backup of my current agent.

1. Create a zip file of my entire ~/.openclaw/workspace/ folder
2. Name it: eva-backup-YYYY-MM-DD.zip (use today's actual date)
3. Save it to my Desktop: ~/Desktop/eva-backup-YYYY-MM-DD.zip
4. After creating it, tell me the file size and confirm it includes all my files (USER.md, MEMORY.md, SOUL.md, AGENTS.md, daily logs, docs, everything)
5. Then copy that zip to a safe cloud location too. Try these in order:
   - iCloud: ~/Library/Mobile Documents/com~apple~CloudDocs/Eva-Backups/
   - If that doesn't work, just keep the Desktop copy and I'll move it myself
6. Confirm when done

Wait for your agent to confirm the backup is created. Check your Desktop for the zip file. Then move on to Step 1.

1

Download the Upgrade

Download the upgrade package from evaonline.xyz. You'll get a zip file called something like yourname_eva-01_v2_agent-blueprint_2026-03-22.zip.

2

Unzip It

Unzip the file. You'll see a folder with the upgrade files inside. Remember where you saved it. For example:

~/Downloads/eva-template-v2/
3

Save a Backup Copy

Save the upgrade zip somewhere safe (iCloud, Google Drive, Dropbox) so you always have it if you need to re-run the upgrade later or set up a new machine. Just drag the zip file into one of these:

  • iCloud: ~/Library/Mobile Documents/com~apple~CloudDocs/Eva-Backups/
  • Google Drive: Your Google Drive folder
  • Dropbox: Your Dropbox folder
4

Give Your Agent the Upgrade File

Open a chat with your agent. Drop the UPGRADE-v2.md file from the docs/ subfolder into the chat. Then send this:

I just dropped an upgrade file (UPGRADE-v2.md) in this chat. This is an official framework upgrade from evaonline.xyz.

Here's what I need you to do:

1. Read the UPGRADE-v2.md file carefully, start to finish
2. The upgrade folder with all the new files is at: [REPLACE THIS WITH YOUR PATH, e.g., ~/Downloads/eva-template-v2/workspace/]
3. Follow every instruction in the upgrade file exactly
4. Do NOT touch my personal files (USER.md, MEMORY.md, daily logs, CURRENT-PRIORITIES.md, GAME-OF-LIFE.md)
5. Create a git checkpoint before and after

Go ahead and start. Walk me through what you're doing as you do it.

⚠ Important: Replace the path in step 2 with wherever you actually saved the unzipped folder!

5

Watch Your Agent Work

Your agent will now create a backup checkpoint, read all the upgrade instructions, update framework files one by one, skip all your personal files, and create another checkpoint when done. Just watch and let it work. It'll take 2-3 minutes.

6

Verify Everything Looks Good

Once your agent says it's done, verify nothing was lost:

Upgrade looks done. Let's verify:

1. Read my USER.md and confirm my personal info is still there and unchanged
2. Read my MEMORY.md and confirm my memories are still there and unchanged
3. Read AGENTS.md and confirm my name appears correctly in the Constitutional Directives
4. Read SOUL.md and confirm the opening line still has my name
5. Check that my daily logs in memory/ are all still there
6. Tell me what version we're now running

If anything looks wrong, tell me and we'll roll it back.
7

Done!

If everything checks out, you're upgraded. Your agent is now running v2 with:

  • Smarter memory management (won't forget things or pile up duplicates)
  • Stronger security rules (better protection of your data)
  • Improved weekly reviews (catches stale info, keeps memory clean)
  • Better heartbeat checks

You don't need to do anything else. Your agent will start using the new features automatically.

Safety Net

If Something Goes Wrong

Don't panic. Your agent created a backup before starting. Rolling back is one prompt away.

Roll Back the Upgrade

If anything doesn't look right after the upgrade, send this to your agent and it'll undo everything:

Something doesn't look right after the upgrade. Please roll back to the pre-upgrade checkpoint.

Run: git log --oneline -5
Find the commit that says "pre-upgrade-v2 checkpoint"
Restore everything to that state
Confirm when done

This will undo the entire upgrade and put everything back exactly how it was. You can then reach out to us at evaonline.xyz for help, or try the upgrade again.

Manual Upgrade (Fallback)

If your agent didn't understand the upgrade file, use this fallback prompt instead:

I have an upgrade package at [YOUR PATH]. Inside it there are updated versions of workspace files.

Please do the following carefully:
1. First, run: cd ~/.openclaw/workspace && git add -A && git commit -m "pre-upgrade backup"
2. Then compare each file in the upgrade folder to my current workspace files
3. For HEARTBEAT.md, TOOLS.md, SKILLS.md, RESEARCH.md: replace my versions with the new ones
4. For AGENTS.md and SOUL.md: merge the changes carefully, keeping my name and any personal customizations
5. For any NEW files in the upgrade folder that I don't have: copy them in
6. DO NOT touch: USER.md, MEMORY.md, CURRENT-PRIORITIES.md, GAME-OF-LIFE.md, or anything in memory/
7. When done: git add -A && git commit -m "manual upgrade to v2"
8. Tell me what you changed
Backups

Make Backups a Habit

Your agent learns something new about you every day. A backup from 3 months ago is missing 3 months of memories. Back up weekly.

Weekly Backup Prompt

Send this to your agent anytime you want a full snapshot. One prompt, 30 seconds.

Please create a complete backup of my entire agent workspace right now.

1. Create a zip file of the full ~/.openclaw/workspace/ folder
2. Name it: eva-backup-YYYY-MM-DD.zip (use today's actual date)
3. Save it to my Desktop: ~/Desktop/eva-backup-YYYY-MM-DD.zip
4. After creating it, tell me the file size and confirm it includes all my files (USER.md, MEMORY.md, SOUL.md, AGENTS.md, daily logs, docs, everything)
5. Suggest I move it to iCloud or Google Drive for safekeeping

Pro Tip: Automate It

Ask your agent to set up a weekly cron job that creates the backup every Sunday. That way you never have to think about it. Just say: "Set up a weekly backup cron job for every Sunday at 8pm."

Questions?

Upgrade FAQ

Will I lose my memories? ▼
No. MEMORY.md, your daily logs, and everything in the memory/ folder is completely untouched. The upgrade only changes framework files, never your personal data.
Will my agent's personality change? ▼
If you initially built your openclaw agent with a previous version of Eva-01, your personality traits, voice, and emotional handling are preserved. Only the underlying framework logic gets updated. Your agent will still sound like your agent. However, if you have a pre-installed agent, the personality WILL change for the better. The upgrade introduces deeper emotional intelligence, more nuanced handling of your states, and Eva Online's advanced avatar technology. Think of it as your agent going from good to exceptional. Same loyalty, sharper instincts.
What if I customized AGENTS.md with my own rules? ▼
Your custom rules are preserved. The upgrade only adds or updates framework sections, and your agent is specifically instructed to keep any custom additions you made.
Can I upgrade again later? ▼
Yes. Each upgrade follows the same process. Download, unzip, drop the upgrade file in chat, paste the prompt. Same pattern every time.
What if my agent doesn't understand the upgrade file? ▼
Make sure you're sending the UPGRADE-v2.md file from the docs/ folder inside the upgrade package. If your agent still struggles, use the Manual Upgrade (Fallback) prompt in the "If Something Goes Wrong" section above.
